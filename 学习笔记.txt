通过B站的【给傻子的Git教程】 https://www.bilibili.com/video/BV1Hkr7YYEh8/?share_source=copy_web&vd_source=6e5b85f7bb3e34f92da9bd2cff0a89ba这个视频学习了git的基本操作，并成功将代码上传到Github

实践流程：先下载git，并配置 user name and email address，后可以创建仓库并将代码上传到本地仓库。
登录Github，配置代理地址，可将代码上传到Github

困难及解决方法：
1.一定要配置 user name and email address
2.上传代码到Github时报错，检查是否配置代理地址，是否因为上传文件过大

心得：
作为一名计算机专业的大学生，我最近初步学习了 Git 的基本使用方法，并成功将本地代码上传到了本地仓库和 GitHub 上，这对我来说是一次非常有收获的学习体验。

起初，我对 Git 的理解仅停留在“版本控制工具”这几个字上，并不了解它的实际用途和重要性。通过本次学习，我系统掌握了 Git 的基本命令，如 git init、git add、git commit、git push 等，并学会了如何在本地创建仓库、提交代码，以及将代码同步到远程仓库 GitHub。特别是在尝试配置 SSH 密钥、处理第一次推送时的身份验证问题等环节，我切实体会到了 Git 的工作机制和它对项目协作管理的巨大帮助。

通过这次学习，我意识到 Git 不仅仅是一个代码管理工具，更是现代软件开发中不可或缺的一部分。它可以帮助开发者高效地管理代码版本，追踪历史更改，回退错误版本，甚至协同多人进行团队开发。尤其是将代码托管到 GitHub 后，不仅可以备份代码，还能展示自己的项目成果，提升个人影响力。

总的来说，这次 Git 的学习让我对开发流程有了更加清晰的认识，也增强了我对今后团队协作开发的信心。我将继续深入学习 Git 的高级用法，例如分支管理、冲突解决和协作流程等，为日后的学习和项目开发打下坚实基础。
